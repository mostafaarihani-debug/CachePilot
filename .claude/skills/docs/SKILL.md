# docs Skill

## Purpose
Help the agent write and maintain clear project markdown files.

## Responsibilities
- Create structured docs.
- Keep instructions concise.
- Maintain consistency across files.
- Split technical, product, and UX content cleanly.
- Update docs when behavior changes.

## Rules
- Use clear headings.
- Keep each file focused on one topic.
- Avoid duplicate instructions across files.
- Prefer actionable language.
- Make files easy for AI and humans to read.

## Recommended doc types
- Product docs.
- Architecture docs.
- UX guidelines.
- Cache explanations.
- Roadmap and testing docs.
- Skill files.

## Success definition
The repo stays easy to understand, easy to extend, and easy for OpenCode to follow.
