# UX_GUIDELINES.md

## Visual style
Use a dark PC-optimizer style interface with a premium utility feel.

## Design principles
- High contrast.
- Clean spacing.
- Rounded cards.
- Soft glow accents.
- Minimal clutter.
- Clear hierarchy.
- Easy-to-scan text.

## Color direction
- Background: very dark charcoal.
- Surfaces: deep gray panels.
- Accent 1: cyan or electric blue.
- Accent 2: green for safe actions.
- Warning: amber.
- Danger: red.
- Text: off-white, not pure white.

## Typography
- Use a modern sans-serif.
- Large readable headings.
- Short body text.
- Strong label/value contrast.
- Avoid dense paragraphs in the UI.

## Layout rules
- Dashboard first.
- Main CTA visible immediately.
- Scan summary above detailed lists.
- Use cards for cache categories.
- Keep settings separate from the main flow.
- Keep side panels narrow and clear.

## Microcopy rules
- Use short sentences.
- Use plain English.
- Avoid “technical-sounding” jargon unless explained.
- Tell the user what a cache is, why it exists, and what happens after cleanup.

## Explanation pattern
For every cache type, show:
- What it is.
- Why it exists.
- When to clean it.
- What may change after cleaning.

## Button rules
- Primary button: "Scan Now" or "Clean Selected".
- Secondary buttons: "View Details", "Exclude", "Learn More".
- Avoid vague labels like "Execute" or "Process".

## Empty states
- Explain what the user should do next.
- Keep them reassuring.
- Avoid blank panels.

## Loading states
- Show progress clearly.
- Tell the user what is being scanned.
- Avoid long silent delays.

## History screen
- Show prior scans.
- Show date and result size.
- Show which categories were cleaned.
- Make it easy to compare scans.

## Logo direction
Use a minimal icon that suggests:
- sweeping/cleaning,
- system health,
- trust,
- speed.

Good shapes:
- shield + sweep arc.
- circular C with sparkle.
- minimal broom + dashboard line.
