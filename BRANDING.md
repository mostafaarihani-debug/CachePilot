# BRANDING.md

## Brand name
CachePilot

## Brand personality
- Modern.
- Trustworthy.
- Calm.
- Technical but friendly.
- Premium without being flashy.

## Brand promise
Help users understand and clean cache safely with one click.

## Visual identity
CachePilot should feel like a polished PC utility, not a generic system tool.

The UI should use dark surfaces, subtle depth, and controlled accent colors so the app feels professional and easy to read.

## Color philosophy
- Avoid pure black backgrounds.
- Avoid pure white text.
- Use dark gray or deep navy surfaces.
- Use off-white text for comfort.
- Limit accent colors to a few purposeful highlights.
- Use color to guide attention, not decorate everything.

## Primary palette

### Backgrounds
- Background 1: `#0F1115`
- Background 2: `#151A21`
- Surface 1: `#1A2028`
- Surface 2: `#212835`
- Elevated surface: `#2A3240`

### Text
- Primary text: `#E8EDF5`
- Secondary text: `#A9B4C2`
- Muted text: `#748294`
- Disabled text: `#55606E`

### Borders and dividers
- Border soft: `#2B3441`
- Border strong: `#3A4656`

### Brand accents
- Primary accent: `#4DA3FF`
- Secondary accent: `#27D3B5`
- Success: `#38D27A`
- Warning: `#F5B84A`
- Danger: `#FF6B6B`

## Accent usage rules
- Use blue for primary actions and focus states.
- Use green for success and safe cleanup confirmation.
- Use amber for warnings and conditional cleanup.
- Use red only for destructive or risky states.
- Never use all accent colors at once.
- Prefer one dominant accent and one support accent.

## UI depth rules
- Use slightly different dark surfaces to separate layers.
- Use shadows and elevation subtly, not heavily.
- Use gradients only in limited places such as headers or hero cards.
- Keep the interface soft and clean, not glossy or neon-heavy.

## Typography rules
- Use a modern sans-serif font.
- Headings should be strong and clear.
- Body text should be short and readable.
- Labels should be concise.
- Avoid all-caps except for small technical tags.

## Logo direction
- Minimal icon.
- Shield, sweep arc, or circular cleanup motion.
- Clean geometric shape.
- Works on dark and light backgrounds.
- Recognizable at small sizes.

## Icon style
- Rounded edges.
- Simple forms.
- High contrast against dark background.
- No complex details.
- No cartoon styling.

## Layout tone
- Spacious.
- Structured.
- Calm.
- High trust.
- Fast to scan.

## Brand voice
- Clear.
- Helpful.
- Honest.
- Simple.
- Non-technical unless explanation is needed.

## Example use cases
- Dashboard cards.
- Scan status.
- Cleanup confirmations.
- Warning banners.
- History timeline.
- Settings panels.

## Accessibility rules
- Maintain strong contrast for text.
- Keep interactive elements easy to identify.
- Do not rely on color alone for meaning.
- Use labels and icons together where needed.

## Design reminders
- Dark does not mean pure black.
- Premium does not mean loud.
- Safe does not mean boring.
- Simple does not mean unfinished.

## Implementation note
Use these colors as design tokens in the app so the theme stays consistent across every screen.
